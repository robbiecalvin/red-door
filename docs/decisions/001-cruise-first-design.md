# ADR 001 — Cruise Mode First

## Decision

Red Door prioritizes Cruise Mode as the primary interaction model.

## Reasoning

Cruise Mode enables spontaneous discovery and map-based interaction between nearby users.

This mode supports:

- real-time social discovery
- location-based interaction
- event coordination

## Consequences

Frontend UI emphasizes map interaction.

Backend services must enforce mode rules server-side.

Date and Hybrid modes remain supported but secondary.
